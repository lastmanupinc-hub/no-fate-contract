Contact and Governance

Overview

No Fate is maintained under open stewardship with transparent governance and community participation. This page provides contact information for governance inquiries, implementation support, and community engagement.

Stewardship

Current Steward

Last Man Up Inc.

Last Man Up Inc. serves as the founding steward of the No Fate ecosystem, responsible for maintaining canonical documents, managing version releases, and coordinating community contributions.

Stewardship Responsibilities

  • Maintaining canonical document integrity and immutability
  • Managing semantic versioning and release process
  • Coordinating community contributions and pull requests
  • Generating and managing cryptographic verification (SHA-256 hashes, GPG signatures)
  • Overseeing test suite and compliance framework updates
  • Facilitating institutional adoption and alignment

Future Governance Model

As part of Phase 4 (Institutional Alignment), stewardship will evolve toward a multi-stakeholder governance model including:

  • Institutional representatives from adopting organizations
  • Academic and research community representatives
  • Implementation specialist representatives
  • Legal domain experts

Governance structure details will be published in 2026.

Contributing to No Fate

Contribution Guidelines

Contributions are welcome for documentation, infrastructure, tools, and community resources. See CONTRIBUTING.md for complete guidelines.

What Can Be Contributed

  • Documentation Improvements: Clarifications, examples, corrections
  • Infrastructure Tools: Test runners, audit tools, verification utilities
  • Reference Implementations: Framework implementations in various languages
  • Playbooks: Domain-specific implementation guides
  • Test Cases: Additional tests for boundary verification
  • Educational Materials: Tutorials, workshops, training resources

What Cannot Be Modified

  • Canonical documents (immutable after publication)
  • SHA-256 hashes
  • GPG signatures
  • Version numbers or tags

Pull Request Process

  1. Fork repository and create feature branch
  2. Make changes following contribution guidelines
  3. Test changes thoroughly
  4. Submit pull request with clear description
  5. Respond to maintainer feedback
  6. Maintainers review and merge

Code of Conduct

All contributors must adhere to community code of conduct:

  • Respectful and professional communication
  • Constructive feedback and collaboration
  • Focus on technical merit and boundary framework principles
  • No harassment, discrimination, or unprofessional behavior

Violations should be reported to no-fate@lastmanupinc.com.

Transparency and Accountability

Public Governance

All governance decisions are documented publicly:

  • Version Releases: Documented in GitHub Releases with rationale
  • Pull Request Review: All reviews conducted publicly in GitHub
  • Issue Tracking: All technical decisions tracked in GitHub Issues
  • Community Discussions: All major decisions discussed openly in GitHub Discussions

Version Creation Process

New version creation follows transparent process:

  1. Proposal published in GitHub Discussions
  2. Community review period (30 days MAJOR, 14 days MINOR)
  3. Feedback incorporation and revision
  4. Draft document published for final review
  5. Maintainer approval with documented rationale
  6. Cryptographic verification generation
  7. Publication with version tag

Audit Trail

Complete audit trail maintained for:

  • All canonical document versions with SHA-256 hashes
  • All GPG signatures with public key fingerprints
  • All version increments with justification
  • All governance decisions with documentation
