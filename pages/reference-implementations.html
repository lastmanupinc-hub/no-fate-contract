Contributing Reference Implementations

Contribution Guidelines

Community contributions of reference implementations are welcome. Implementations should:

  • Follow Three-Layer Model: Implement boundary detection, refusal enforcement, and audit/verification
  • Pass Test Suite: Score minimum 30/34 on Deterministic Boundary Test Suite
  • Include Documentation: Usage examples, API documentation, integration guides
  • Provide Tests: Unit tests and integration tests demonstrating compliance
  • License Appropriately: Use permissive license (MIT, Apache 2.0) for implementations

Submission Process

  1. Develop implementation following Developer Integration Guide
  2. Run complete Deterministic Boundary Test Suite and document results
  3. Create repository with comprehensive documentation
  4. Submit pull request to No Fate repository adding your implementation to catalog
  5. Maintainers will review for compliance and quality

Implementation Requirements

  • Must implement all six refusal types
  • Must support partial determinism handling
  • Must include audit logging
  • Must document which No Fate version implemented (e.g., v1.0.0, v1.1.0)
  • Must include SHA-256 verification of canonical documents used

Implementation Certification

Certification Process (Planned)

Formal certification program for reference implementations is planned for Phase 4. Certification will require:

  • Score 32/34 or higher on Deterministic Boundary Test Suite
  • Pass manual audit using 34-point audit template
  • Independent verification by qualified auditors
  • Documentation review
  • Security review

Self-Assessment

Until formal certification available, implementations can self-assess using:

  • Deterministic Boundary Test Suite v1.0
  • Compliance Guide v1.0.0 audit template

Document self-assessment results and include in implementation repository.
